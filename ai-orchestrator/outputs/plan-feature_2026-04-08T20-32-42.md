## Execution Plan

### Phase 0 — Architect Review (sequential, önce)

**Kararlar:**

1. **Status Transition Güvenliği**
   - Clothing: `Created → Delivered` direkt geçiş domain'de izin verilmeli. Mevcut `ChangeStatus` muhtemelen sıralı geçiş zorluyor → yeni `SkipToDelivered()` veya `ChangeStatus`'a Clothing bypass eklenmeli
   - Mevcut state machine kırılmamalı — Catering akışı aynen korunmalı

2. **Command Orchestration Paterni**
   - MarkReadyCommand içinden ExportShipmentToNetsisCommand çağrısı: MediatR `ISender.Send()` ile internal dispatch (aynı transaction scope'ta değil — Netsis çağrısı external, fail ederse shipment yine ReadyForDispatch olmalı)
   - Önerilen pattern: Handler içinde `try { await _sender.Send(exportCmd); } catch { warnings.Add(...); }` — non-blocking

3. **OperationType Propagation**
   - `ShipmentDto` ve `ShipmentListItemDto`'ya `OperationType` eklenmeli
   - Kaynak: `Shipment.Project.OperationType` → query'lerde Include veya projection
   - Bu DTO değişikliği tüm frontend'i etkiler, breaking change değil (additive)

4. **Irsaliye Auto-Fetch Timing**
   - Netsis sipariş oluşturduktan hemen sonra irsaliye kesilmemiş olabilir → auto-fetch best-effort, sessiz fail
   - Retry mekanizması bu scope'ta YOK — manuel buton yeterli

5. **Reconciliation Query Stratejisi**
   - Yeni DbContext query (read-only) — ayrı handler, projection ile
   - NetsisQty computed: OperationType'a göre DeliveredQty veya IssQty
   - Status enum backend'de hesaplanır, frontend sadece render eder

6. **Transaction Boundaries**
   - Catering export (non-blocking): Shipment save → Netsis call → success ise NetsisTransferredAt update (ikinci save veya aynı UoW)
   - Clothing export (blocking): Netsis call → success ise status + NetsisTransferredAt → tek save
   - Irsaliye fetch: her zaman ayrı, lightweight

#### Geliştirme 4: Reconciliation

**Yeni query: `GetReconciliationQuery`**
- Params: `DateTime FromDate, DateTime ToDate, bool OnlyDiff, OperationType? OperationType`
- Shipment'ları filtrele: NetsisTransferredAt BETWEEN fromDate AND toDate
- Projection:

```csharp
public class ShipmentReconciliationDto
{
    public int ShipmentId { get; set; }
    public string TalepNo { get; set; }
    public string ProjectCode { get; set; }
    public string ProjectName { get; set; }
    public string OperationType { get; set; } // "Catering" / "Kıyafet"
    public DateTime? NetsisTransferredAt { get; set; }
    public string IrsaliyeNo { get; set; }
    public DateTime? IrsaliyeDate { get; set; }
    public bool HasDifference { get; set; }
    public List<LineReconciliationDto> Lines { get; set; }
}

public class LineReconciliationDto
{
    public string StockCode { get; set; }
    public string StockName { get; set; }
    public string NetsisStockCode { get; set; }
    public decimal IssQty { get; set; }        // IssOrderLine?.Miktar ?? OrderedQty
    public decimal OrderedQty { get; set; }
    public decimal DeliveredQty { get; set; }
    public decimal NetsisQty { get; set; }      // Computed: Catering→DeliveredQty, Clothing→IssQty
    public string Status { get; set; }          // enum string
}
```

- Status hesaplama logic:
  ```
  if (operationType == Clothing) → "ClothingDirect"
  else if (IssQty == OrderedQty == DeliveredQty == NetsisQty) → "Equal"
  else if (IssQty != OrderedQty) → "Revised"
  else if (DeliveredQty == 0) → "NotPicked"
  else if (DeliveredQty < OrderedQty) → "UnderPicked"
  else if (DeliveredQty > OrderedQty) → "OverPicked"
  ```
  - NOT: Birden fazla durum olabilir (Revised + UnderPicked). Öncelik: NotPicked > Revised > UnderPicked > OverPicked > Equal

- `OnlyDiff == true` → `HasDifference == true` olanları filtrele
- `HasDifference` = Lines.Any(l => l.Status != "Equal" && l.Status != "ClothingDirect")

**Endpoint:**
```
[HttpGet("reconciliation")]
[Authorize(Roles = "Admin,Manager,Accounting")]
```
